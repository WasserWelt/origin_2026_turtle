### 步骤 6：底盘板 — `bsp_can_chassis.c` (`chassis_board/Chassis_Board/BSP/bsp_can_chassis.c`)

#### 6.1 接收解析分支调整 (lines 150-265)

**移入 CAN1 接收分支的 case：**

- `GIMBAL_TO_CHASSIS_FIRST_RecID (0x10)` — 从 CAN2 移入 CAN1
- `GIMBAL_TO_CHASSIS_SECOND_RecID (0x11)` — 从 CAN2 移入 CAN1

**保留在 CAN2 的 case：**

- `WHEEL1-4_M3508_RecID`
- `CAP_RecID`
- `POWER_METER_RecID`

**CAN1 接收分支整理后结构：**

```c
if (hcan == &hcan1)
{
    HAL_CAN_GetRxMessage(hcan, CAN_RX_FIFO0, &rx_header, rx_data);
    switch (rx_header.StdId)
    {
    case BIG_YAW_DM6006_RecID:   // 0x300 — 来自云台板转发的大Yaw位置
    {
        DM_big_yaw_motor.id = (rx_data[0]) & 0x0F;
        DM_big_yaw_motor.state = (rx_data[0]) >> 4;
        DM_big_yaw_motor.p_int = (rx_data[1] << 8) | rx_data[2];
        break;
    }
    case GIMBAL_TO_CHASSIS_FIRST_RecID:   // 0x10 — 上下板通讯 RC数据
    {
        uint8_t rc_connected;
        chassis_rc_ctrl.s[1] = rx_data[0];
        rc_connected = rx_data[1];
        chassis_rc_ctrl.ch[2] = (rx_data[2] << 8) | rx_data[3];
        chassis_rc_ctrl.ch[3] = (rx_data[4] << 8) | rx_data[5];
        chassis_rc_ctrl.ch[4] = (rx_data[6] << 8) | rx_data[7];
        if (rc_connected)
            detect_hook(RC_TOE);
        break;
    }
    case GIMBAL_TO_CHASSIS_SECOND_RecID:  // 0x11 — 上下板通讯导航数据
    {
        int nav_vx_int = (rx_data[1] << 8) | rx_data[0];
        int nav_vy_int = (rx_data[3] << 8) | rx_data[2];
        nav_ctrl.vx = uint_to_float(nav_vx_int, NAV_MIN_SPEED, NAV_MAX_SPEED, 12);
        nav_ctrl.vy = uint_to_float(nav_vy_int, NAV_MIN_SPEED, NAV_MAX_SPEED, 12);
        nav_ctrl.chassis_target_mode = rx_data[4] & 0x03;
        nav_ctrl.updownhill_state = (rx_data[4] >> 2) & 0x03;
        nav_ctrl.health_state = (rx_data[4] >> 4) & 0x01;
        nav_ctrl.buffer_energy_remain = ((rx_data[5] & 0x07) << 3) | (rx_data[4] >> 5);
        nav_ctrl.referee_power_limit = ((rx_data[6] & 0x07) << 5) | (rx_data[5] >> 3);
        nav_ctrl.game_start = (rx_data[6] >> 3) & 0x01;
        detect_hook(NAV_TOE);
        break;
    }
    case POWER_METER_RecID:  // 0x123 — 功率计反馈（从CAN1接收）
    {
        real_power = EnergyDataToFloat(rx_data);
        break;
    }
    }
}
```

#### 6.2 CAN2 接收分支调整 (lines 199-265)

**删除** `GIMBAL_TO_CHASSIS_FIRST_RecID` 和 `GIMBAL_TO_CHASSIS_SECOND_RecID` case（已移至 CAN1）。

#### 6.3 CAN2 过滤器 `Can_Filter_Init` 修改 (lines 79-91)

```c
can_filter.FilterIdHigh = (GIMBAL_TO_CHASSIS_FIRST_RecID << 5);   // 0x10
can_filter.FilterIdLow  = (GIMBAL_TO_CHASSIS_SECOND_RecID << 5);  // 0x11
can_filter.FilterMaskIdHigh = (WHEEL1_M3508_RecID << 5);          // 0x201
```


## 五、一个需要确认的问题

**关于大 Yaw (DM6006) 位置数据传递给底盘的方式：**

当前设计中，底盘板通过 CAN1 接收 `0x300`（DM6006 电机原始反馈 ID）。但云台板和底盘板的 CAN2 并不相连（只有 CAN1 相连）。

需要确认：**大 Yaw 电机反馈 (0x300) 是通过什么物理路径传递给底盘板的？**

两种方案：

1. **方案 A**：在云台板 `Send_Chassis_Task` 中新增一帧 (`CAN_GIMBAL_TO_CHASSIS_THIRD_ID`, 0x101)，将 DM6006 的 `p_int` 数值转发到底盘板，底盘板从 CAN1 接收后用于跟随计算
2. **方案 B**：底盘板的 CAN1 过滤器接收 0x300，但这需要底盘 CAN1 和云台 CAN2 物理相连（当前未相连）

如果使用**方案 A**，则需要补充 `Send_Chassis_Task.c` 中 `CAN_GIMBAL_TO_CHASSIS_THIRD_CMD` 的实际发送代码。是否需要我将此方案加入计划？

回答：**大 Yaw (DM6006) 的反馈云台板和底盘板都能直接收到，因为他们在同一条can总线上。**
